## 3. Diskussionsfragen

### Schmerzpunkte heute
- Welche Informationen sucht ihr am häufigsten zusammen, bevor ihr eine Entscheidung treffen könnt?
- Wo geht die meiste Zeit verloren — Daten sammeln, Daten abgleichen, oder Daten interpretieren?
- Welche Fehler passieren, weil Informationen nicht aktuell oder nicht zugänglich waren?

### Prozess-Phasen
- In welcher Phase des P2P-Zyklus wäre der Companion am wertvollsten?
  - [ ] Bedarfsermittlung & Forecast
  - [ ] PO-Erstellung & Freigabe
  - [ ] Produktionsmonitoring
  - [ ] Inbound & Wareneingang
  - [ ] Bestandsmanagement & Replenishment
- Gibt es saisonale Unterschiede? (z.B. Order-Phasen Winter Dez–Feb vs. Normalphase)

### Quellen & Aktualität
- Welche eurer Google Sheets wären die wichtigsten Datenquellen für Sektionen?
- Gibt es Informationen, die nur in E-Mails oder Köpfen existieren und die systematisch erfasst werden sollten?
- Wie aktuell müssen die Daten sein? (Echtzeit vs. täglicher Snapshot vs. wöchentliches Update)

### Team & Rollen
- Sollten alle 7 SCM-Teammitglieder eigene Profile bekommen?
- Gibt es Rollen, die das gleiche Profil teilen können? (z.B. Max und Amelie bei Replenishment?)
- Wie geht ihr mit Vertretungen um? (Soll jemand temporär das Profil eines Kollegen laden können?)

### Proaktive Features
- Sollte der Agent **proaktiv warnen**, wenn:
  - [ ] Bestandsreichweite unter Schwellenwert fällt
  - [ ] PO hinter Plan liegt
  - [ ] Lieferant Performance-Probleme zeigt
  - [ ] Materialverzögerung eine Produktion blockiert
  - [ ] Übergabe aus PD blockiert ist
- Oder soll der Agent nur antworten, wenn gefragt?

### ERP-Übergang
- Wenn Odoo live geht, welche Sektionen profitieren am meisten von ERP-Daten?
- Welche Sektionen bleiben auch mit ERP auf manuelle/Meeting-basierte Quellen angewiesen?

## 4. Offene Design-Fragen

| Frage | Optionen | Eure Meinung |
|-------|---------|-------------|
| "SCM Essentials"-Sektion für alle? | Ja (Basiswissen für jeden im Team) / Nein (alles rollenspezifisch) | |
| Profil-Wechsel erlauben? | Fest pro Person / Temporäre Overrides (z.B. Order-Phase) | |
| Kontext-Transparenz? | Agent zeigt geladenen Kontext / Unsichtbar | |
| Proaktive Warnungen? | Ja, bei kritischen Schwellenwerten / Nein, nur auf Nachfrage | |
| Wer pflegt Sektionen? | Automatisch + Owner pro Sektion / Zentral durch eine Person | |
| Cross-Team-Sektionen? | Marketing-Sektionen als Hintergrund für SCM verfügbar? | |
